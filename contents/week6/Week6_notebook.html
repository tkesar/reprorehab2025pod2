Control Flow & Functions¶

Why this matters?¶

Most real programs are: decisions + repetition + reusable building blocks.

This file focuses on:

  • Decisions: if/elif/else, match/case (and why match/case can be clearer than long if/elif chains)
  • Repetition: for/while, break/continue, enumerate, zip (tuple-unpacking)
  • Reuse: functions (parameters, defaults, docstrings, type hints, multiple returns)

Notes for learners¶

  • elif must immediately follow the if it belongs to (only comments/blank lines may be in between).
  • Truthiness: 0, 0.0, '', [], {} are False; non-zero numbers and non-empty containers are True. Try: print(bool(0), bool(2)) to see it.
  • Prefer clarity over cleverness. We'll show both clear and compact patterns when helpful.
  • Safety: For while True loops, we include a small safeguard counter to avoid accidental infinite loops.
  • Python 3.10+ is assumed for str | None type hints and match/case. On 3.9 use Optional[str] and if/elif.

if/elif/else¶

Syntax:

if (condition1):
  do action1         # executed if 'condition1' == True
elif (condition2):
  do action2
...
else:
  do action{N}       # action{N} with condition1, condition2, ..., condition{N-1} all being False

# `isinstance()` is a function that checks if an input is equal to a type of

if/else in list comprehension¶

  • Binary choice (only consider True cases):
    [(action) for x in {iterable} if (condition)]
  • Additional treatment specified:
    [(action1) if (condition) else (action2) for x in {iterable}]
In [2]:
# Only consider True (i.e. non-zero numbers) cases
integers = range(10)
# Odd numbers or even numbers?
odd_or_even = [x for x in integers if x % 2]

% is a modulo operator in Python (ex. 3 % 2 = 1).

  1. {iterable} is range(10). This means x will be 0, 1, 2, ... 9 at each iteration
  2. (action) is x, which means to append x to a list.
  3. (condition) is x % 2. At each iteration, if this is True, append x.

In boolean indexing, 0 is False, and any non-zero number is True (yes, not only 1, but 2, 3, etc.).

  • If x is even, then x % 2 == 0 (False) -> x not appended to the list.
  • If x is odd, then x % 2 == 1 (True) -> x appended to the list.

elif is not directly used in a list comprehension. Instead, you 'nest' an if-else condition inside an else condition.

  • Typical if/elif/else statement:
    a_list = list()
for x in {iterable}:
  if (condition1):
    a_list.append(action1)
  elif (condition2):
    a_list.append(action2)
  else:
    a_list.append(action3)
  • List comprehension's case:
    a_list = [(action1) if (condition1) else (action2) if (condition2) else (action3) for x in {iterable}]

break/continue¶

break will break the loop. continue is to move to the next iteration without any action performed.

In [13]:
# break a loop in the middle
# `first_names` has 48 elements, so `i` can iterate over range(48)
for i, name in enumerate(first_names):
  if name == "Nancy":
    print(f"Nancy found! Closing the loop at the iteration: {i}")
    # iteration terminates when `name == "Nancy"`
    break
  else:
    # If `name == "Nancy"` is False, do nothing and move to the next iteration.
    continue

Nancy found! Closing the loop at the iteration: 40

Use while when you don't know the number of iterations needed.¶

for loops will end when it reaches the last iteration index. If you need to loop while a condition is True, use while.

In [14]:
# Let's suppose you don't know the index of the first name "Lucretia" in the list: `first_names`

to_lookfor = 'Lucretia'
i = 0   # A start index is typically set to 0 - the starting index
while first_names[i] != to_lookfor:
  i += 1    # `i += 1` is the shorthand of `i = i + 1`

# Remember, Python starts counting from 0, not 1
print(f"{to_lookfor} is the {i+1}th first name")

Lucretia is the 21th first name

# While loop can be "broken" as well

k = 0
counter = 0
# Don't do this - it can lead to an infite loop...
while True:
  if first_names[k] == "Martha":
    counter += 1
  # If "Martha" is found twice, break the loop
  if counter == 2:
    break
  k += 1

You may want to provide a safe guard to avoid infinite loops if your code is not correctly written.

max_iter = 1000  # limit the number of iterations
steps = 0

while True:
    ...  # whatever action you want to loop over
    steps += 1  # increase the counter
    # if the loop iterates over `max_iter` times,
    # it will raise an error:
    if steps > max_iter:
        raise RuntimeError("Possible infinite loop-check your logic.")

If you have convoluted / nested logics, use for loops. Whenever possible, however, use comprehensions or even vectorized methods provided by numpy or pandas.

Your code may look longer, but it will be faster if your data is BIG.

#@title Benchmark
from time import time

def loop_performance(x, iter=100):
  op_times = list()
  for _ in range(iter):
    tic = time()
    acc_rms_for = list()
    for acc in zip(x[0], x[1], x[2]):
      acc_rms_for.append(float(np.sqrt(acc[0]**2 + acc[1]**2 + acc[2]**2)))
    op_times.append(time() - tic)
  return (np.mean(op_times), np.std(op_times))

def numpy_performance(x, iter=100):
  op_times = list()
  for _ in range(iter):
    tic = time()
    acc_rms_np = np.linalg.norm(x, axis=1)
    op_times.append(time() - tic)
  return (np.mean(op_times), np.std(op_times))

Functions¶

Start when you see typing the same operation again and again.

At some point, you will start by preparing bunch of functions before conducting an actual analysis (modularization).

The structure of a function is this:

def some_function(parameter):
  what a function does

In [24]:
# Function parameters can have default values:
# Here, default values of the specified by:
#   `(name of a parameter)=(default value)`
# So here, the default of `num1` is `None`. The same is true for `num2`.
def add_nums(num1=None, num2=None):
  # `all(iterable)` returns True if everything in iterable is True.
  # So here, if `num1` is None and `num2` is None, it will be True
  if all((num1 is None, num2 is None)):
    # If conditional is True, return `None`.
    # A function 'ends' with a `return` command.
    return None
  # This is why you don't need explicit `else` statement here.
  # If `if` statement above is true, there's already an associated `return` statement,
  # so the function will no longer proceed. If `if` statement is not true,
  # following `return` statement will run.
  return num1 + num2

# This returns None, because no `num1` or `num2` value is provided.
# Both parameters will be `None`, and the function will return `None`
#   -> so you don't see anything returned
add_nums()

# If `num1` is not None or `num2` is not None, `num1 + num2` is returned.
add_nums(3, 7)
Out[24]:
10

There's one more concept here: type hints. Type hints are annotations that specify the expected data types for variables and function parameters, improving code readability and helping to catch errors before runtime.

Type hints are provided using :.

def some_function(param:str):
    ...
    return ...

Now the function some_function will expect a parameter param to be a string. However, even if you provide a different data type, it will still proceed. Again, type hints are mostly for readability.

Docstring¶

A docstring is a string literal that occurs as the first statement in a module, function, class, or method definition.

Use triple double quotes (""") around a docstring.

  • Single line docstring: """this is a docstring."""
  • Multiple line docstring:\
    """
This is a multi-line docstring.
Right?
Yeah right.
"""

    -----
    Arithmetic is modular when using integer types, and no error is
    raised on overflow.

    The sum of an empty array is the neutral element 0:

    >>> np.sum([])
    0.0

    For floating point numbers the numerical precision of sum (and
    ``np.add.reduce``) is in general limited by directly adding each number
    individually to the result causing rounding errors in every step.
    However, often numpy will use a  numerically better approach (partial
    pairwise summation) leading to improved precision in many use-cases.
    This improved precision is always provided when no ``axis`` is given.
    When ``axis`` is given, it will depend on which axis is summed.

    more precise approach to summation.

Nested function / inner function¶

You can define a function inside a function. Of course, it will be local to the outermost function.

In [32]:
def nested_function(val:list|None=None):
  """
  nested_function will show how you can nest a function
  inside another function.

  If `val` is not a list, return None.
  Otherwise, return a new list with each element doubled.

  Parameters
  ----------
  val: list|None (this means the expected input for `val` is either a list or None)
    a list to process; default to None

  Returns
  -------
  None if val is None, double_it(val) otherwise.
  """
  if not isinstance(val, list):
    return None

  def double_it(to_double:list):
    """Double the elemenfs of a provided list"""
    return [x * 2 for x in to_double]

  return double_it(val)

def ecdf(ax, x, hue, data, xlabel, title, palette='deep'):
    """
    ecdf will prepare an empirical cumulative distribution function (ECDF)
    plot on the provided axis.

    Parameters
    ----------
    ax: matplotlib.axes.Axes
      axis to plot on
    x: str
      column name for x-axis variable
    hue: str
      column name for hue variable
    data: pd.DataFrame
      data source for the plot
    xlabel: str
      label for x-axis
    title: str
      title for the plot
    palette: str   
      color palette for the plot; default to 'deep'
    """
    # For more details of `sns.ecdfplot`, check:
    # https://seaborn.pydata.org/generated/seaborn.ecdfplot.html
    sns.ecdfplot(data=data, x=x, hue=hue, ax=ax, palette=palette,
    linewidth=1.8, hue_order=["Female", "Male"])
    ax.set_xlabel(xlabel, fontsize=11)
    ax.set_ylabel('ECDF', fontsize=11)
    ax.tick_params(axis='both', which='major', labelsize=8)
    ax.set_title(title, loc="left", weight="bold", fontsize=11)

# preparing a 10x6 inch figure with 2 rows and 3 columns of subplots
# hspace and wspace are the height and width spaces between subplots
fig = plt.figure(figsize=(10, 6))
gs = GridSpec(2, 3, figure=fig, hspace=0.4, wspace=0.3)

# We are making `axes` dictionary to map each column name to each subplot axis.
# This way, we can loop through `axes` dictionary later.
keys = kin_df.columns[2:]
axes = {
    keys[0]: fig.add_subplot(gs[0, 0]),
    keys[1]: fig.add_subplot(gs[0, 1]),
    keys[2]: fig.add_subplot(gs[0, 2]),
    keys[3]: fig.add_subplot(gs[1, 0]),
    keys[4]: fig.add_subplot(gs[1, 1]),
    keys[5]: fig.add_subplot(gs[1, 2]),
}

# 1. Prepare titles and xlabels separately
titles = [ "1. Movement rate", "2. Peak acceleration",
"3. Average acceleration", "4. Movement duration",
"5. Sample Entropy (pk)", "6. Sample Entropy (avg)"]

xlabels = ["Movement rate (mov/h)", "Peak acceleration (m/s²)",
"Average acceleration (m/s²)", "Movement duration (s)",
"Sample Entropy (peak accel.)", "Sample Entropy (avg. accel.)"]

# 2. Loop through axes and titles using `zip`
for key, title, xlabel in zip(keys, titles, xlabels):
    ecdf(ax=axes[key], x=key, hue="sex", data=kin_df, xlabel=xlabel, title=title)

# This command will remove the top and the right spines of all subplots
sns.despine(fig=fig)

So you see that the you can use a function instead of writing multiple lines that are basically identical, differing only in their parameter values. Using for-loop to iterate over titles to labels is another key point to note.

Of course, this may limit your ability to customize each subplot, since the same style will be applied across all of them. For example, if you want to adjust the x-axis limit of subplot 5 only, you'll need to make additional tweaks. Still, this is an effective way to automate your workflow!
